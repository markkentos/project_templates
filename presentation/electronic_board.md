# Заготовка для электронной доски (Whiteboard / Miro Layout)

Этот документ содержит концепт и структуру интерактивной электронной доски (Miro, Padlet или классическая маркерная доска) для наглядной демонстрации проекта **«Anime Shelf»** во время защиты.

---

## 🎨 Общая концепция доски
Доска разделена на 4 смысловые зоны (фрейма), которые спикеры открывают и комментируют по ходу презентации. Связи между блоками обозначаются цветными стрелками.

---

## Фрейм 1: Пользовательский интерфейс и Клиентский каталог
*Центральная визуальная часть, показывающая как пользователь взаимодействует с системой.*

```
+---------------------------------------------------------------------------------+
|                                 КЛИЕНТСКАЯ ЧАСТЬ                                |
|                                                                                 |
|  [Каталог товаров (/)]  === (запрос) ===>  [ProductCatalogProxy]                |
|           ||                                     ||                             |
|     (рекомендации)                       (фильтрация и кэш)                     |
|           ||                                     ||                             |
|  [Адаптер аниме-трендов]                  [ProductCatalogService]               |
|           ||                                     ||                             |
|  [Legacy Trend API (JSON)]               [Категории и Товары в БД]              |
|                                                                                 |
|  [Дерево категорий (/tree/)]                                                    |
|   - CategoryNode (Компоновщик)                                                  |
|   - ProductLeaf (Лист) ===== [CatalogIterator (Итератор)] ===> [Вывод в UI]     |
+---------------------------------------------------------------------------------+
```

### Цвета стрелок:
*   **Синие стрелки**: Поток запросов и ответов каталога.
*   **Оранжевые стрелки**: Преобразование данных адаптером трендов.

---

## Фрейм 2: Жизненный цикл заказа (Command -> State -> Observer)
*Демонстрация поведенческих паттернов в действии при покупке.*

```
+---------------------------------------------------------------------------------+
|                             ЦЕПОЧКА ОБРАБОТКИ ЗАКАЗА                            |
|                                                                                 |
|  [Кнопка "Оформить"] ===> [CheckoutCommand (Команда)]                           |
|                                     ||                                          |
|                               (запускает)                                       |
|                                     \/                                          |
|                       [OrderProcessTemplate (Шаблонный метод)]                  |
|                       - StandardOrderProcess / GiftOrderProcess                 |
|                                     ||                                          |
|                           (меняет статус заказа)                                |
|                                     \/                                          |
|                       [OrderStateMachine (Машина состояний)]                    |
|                Created -> Paid -> Packed -> Shipped -> Delivered                |
|                                     ||                                          |
|                           (генерирует событие)                                  |
|                                     \/                                          |
|                            [EventBus (Издатель)]                                |
|                               /     |      \                                    |
|                      (оповещает зарегистрированных наблюдателей)                |
|                             /       |        \                                  |
|                            \/       \/       \/                                 |
|                  [ProcessLog]  [LowStock]  [Notification]                       |
|                   (Логист)      (Склад)     (Уведомления)                       |
+---------------------------------------------------------------------------------+
```

### Интерактивные стикеры на доске:
*   📌 **Стикер 1 (Команда)**: «Отрезает контроллер от БД. Views знают только класс команды!»
*   📌 **Стикер 2 (Состояние)**: «Нельзя перевести заказ из Создан сразу в Отправлен. Защита от дурака!»
*   📌 **Стикер 3 (Наблюдатель)**: «Слабая связанность. Если нужно слать Telegram-уведомления, просто добавляем новый класс-подписчик!»

## Фрейм 3: Математическая модель аналитики спроса
*Визуализация алгоритма регрессии.*

```
+---------------------------------------------------------------------------------+
|                       МАТЕМАТИЧЕСКИЙ ПРОГНОЗ СПРОСА И ДЕФИЦИТА                   |
|                                                                                 |
|  Недели продаж (SalesPoint):  [Нед 1: 3 шт] -> [Нед 2: 5 шт] -> [Нед 3: 7 шт]   |
|                                        ||                                       |
|                                  (ввод данных)                                  |
|                                        \/                                       |
|                               [Линейная регрессия]                              |
|                                     y = ax + b                                  |
|                                        ||                                       |
|                               (расчет коэффициентов)                            |
|                                        \/                                       |
|                         Прогноз спроса на Неделю 4: 9 шт                        |
|                                        ||                                       |
|                               (сопоставление)                                   |
|                                        \/                                       |
|                  Складской остаток (Предложение): 10 шт                         |
|                                        ||                                       |
|                               (вывод на экран)                                  |
|                                        \/                                       |
|                 Результат: "Предложения хватает" (Разница +1 шт)                |
+---------------------------------------------------------------------------------+
```

### Графическое пояснение для комиссии:
*   Ось $X$ — Номера недель, Ось $Y$ — Количество продаж.
*   Строится прямая линия тренда, идущая вверх (рост спроса) или вниз (падение спроса).
*   Горизонтальная линия — уровень складских остатков. Точка пересечения или разница показывает дефицит.

## Фрейм 4: Карта паттернов и Фасад (Итог защиты)
*Единая архитектурная картина.*

```
+---------------------------------------------------------------------------------+
|                               ЕДИНАЯ ТОЧКА ВХОДА                                |
|                                                                                 |
|                            [views.py (Контроллеры)]                             |
|                                       ||                                        |
|                             (вызывает исключительно)                            |
|                                       \/                                        |
|                              [StoreFacade (Фасад)]                              |
|           /             /             |             \             \             |
|          \/            \/            \/             \/            \/            |
|      [Proxy]       [Adapter]     [Commands]     [Composite]   [Strategies]      |
|     (Каталог)     (Тренды)       (Действия)     (Категории)   (Цены/Доставка)   |
+---------------------------------------------------------------------------------+
```

### Главный вывод на доске:
> 🚀 **«Паттерн Фасад связывает все 12 паттернов в единую монолитную, но слабо связанную систему. Веб-интерфейс изолирован от бизнес-правил, что гарантирует легкость сопровождения и тестирования приложения!»**
